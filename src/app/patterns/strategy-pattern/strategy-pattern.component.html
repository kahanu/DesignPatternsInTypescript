Strategy Pattern

by King Wilder
The strategy pattern (also known as the policy pattern) is a behavioral software design pattern that enables selecting an algorithm at runtime. Instead of implementing a single algorithm directly, code receives run-time instructions as to which in a family of algorithms to use.

This pattern is useful if you find yourself in a situation where you are using either a switch statement or if/else with many branches. It's best to break that functionality out into separate classes that handle just the logic for that strategy.

Temperature Converter

In this sample application, we will convert temperatures from Fahrenheit to Celsius to Kelvin.

In the example with the pattern, I encapsulate the formulas for each conversion in it's own class, that is composed into a conversion class. There is more code in the Strategy version, but in larger applications the strategy pattern helps to make ongoing management of the application easier.
