State Pattern

by King Wilder
The state pattern is a behavioral software design pattern to allow an object to alter its behavior when its internal state changes.

When is this used?

The State Pattern can be used anywhere you need to change the state of an object and react to it. Your cellphone has a state machine that changes the state of it's connection based on your coverage area. When the state changes in your application, you can react to it, by preventing a specific action, and alerting the user of a problem or invalid command.

Shopping Cart

This demonstration will use the shopping cart as the example of how to use the State Pattern. Each step of the shopping cart path is a different state, and the application keeps track of the current state, and can allow you to proceed or go back. When it hits specific states, like the end of the cart process, it can alert the application to change the buttons so the user cannot proceed, but instead continue shopping.
